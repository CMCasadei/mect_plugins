Description de la Crosstable

La Crosstable est divisée en sections, certaines disponibles pour l'utilisateur et d'autres bloquées et pré-compilées avec les variables de l'appareil.

Section “Retentive”, les variables de cette section ne perdent pas leur valeur après avoir éteint et rallumé l'appareil Mect.

Section “Non Retentive”, grande section disponible pour toutes les variables de votre projet qui n'ont pas besoin d'être “mémorisées” après avoir éteint l'appareil.

Les sections “Diagnostic”, “Local I/O” et “System” peuvent être partiellement modifiées par l'utilisateur et peuvent être utilisées dans vos projets , elles vous permettent non seulement de déplacer les E/S, mais fournissent également des informations importantes sur le fonctionnement interne de l'appareil Mect.


Champ

Description

Priority

Permet l'échange de données avec des appareils distants.

- 0: Non activé ;

- 1: Activé avec priorité élevée ; de préférence pour les variables avec protocole “PLC”

- 2: Activé avec priorité moyenne ; de préférence pour les variables avec protocole “RTU”

- 3: Activé avec une faible priorité; de préférence pour les variables avec protocole “TCP”

Update

Définit le type d'échange de données activé avec les appareils distants.

-H: La variable n'est mise à jour que si elle est affichée sur la page courante;

-P: La variable est toujours mise à jour;

-S: la variable est toujours mise à jour et stockée lentement avec un temps “Slow Log Period” (paramètre System Editor);

-F: comme “S” mais rapidement et avec un temps “Fast Log Period” (paramètre System Editor);

-V: la variable n'est acquise et historisée que lorsqu'il y a un changement;

-X: La variable est historisée lorsqu'elle est appelée par la fonction logShot().

Var.Name

Nom de la variable. Il ne doit pas comporter plus de 30 caractères.

Type

Les types suivants sont définis:

-BIT: Booléen;

-BYTE: octet;

-BYTE_BIT: sélectionne un bit dans un octet;

-WORD_BIT: sélectionne un bit dans un mot (16 bits);

-DWORD_BIT: sélectionne un bit dans un mot double (32 bits);

-UINT: entier sans signe 16 bits;

-UINTBA: entier sans signe 16 bits avec octets inversés;

-INT: entier avec signe 16 bits;

-INTBA: entier avec signe 16 bits avec octets inversés;

-UDINT: entier sans signe 32 bits;

-UDINT....: entier sans signe 32 bits avec octets inversés;

-DINT: entier avec signe 32 bits;

-DINT....: entier avec signe 32 bits avec octets inversés;

-REAL: réel;

-REAL...: réel avec octets inversés.

Decimal

Pour les variables entières ou réelles, il indique le nombre de décimales après la virgule avec laquelle la variable est représentée. Dans le cas où la variable est du type BYTE_BIT, WORD_BIT ou DWORD_BIT indique la position du BIT à partir de 1.

Protocol

Indique le protocole avec lequel la variable est échangée:

-PLC: variable d'échange entre PLC et IHM

-RTU: variable présente sur un appareil distant via Modbus RTU

-TCP: variable présente sur un appareil distant via Modbus TCP

-TCPRTU: variable présente sur un appareil distant via Modbus TCPRTU

-CANOPEN: variable présente sur un dispositif CANopen distant

-RTU_SRV: variable répondant à une requête d'un master Modbus RTU

-TCP_SRV: variable répondant à une requête d'un master Modbus TCP

-TCPRTU_SRV: variable répondant à une requête d'un master Modbus TCPRTU

-MECT: variable sur protocole MECT

IP Address

Adresse IP pour Modbus TCP et TCPRTU, par exemple : 192.168.0.42

Port

Dans le cas de variables NON PLC, indique le port de communication à utiliser : le port de communication dépend de l'appareil utilisé.

Node ID

Adresse du nœud de l'appareil distant. Dans le cas de l'adresse 0, l'écriture est de type Broadcast.

Register

Adresse Modbus du registre auquel on veut accéder.

Block

Adresse du premier registre de blocs. Pour les registres qui font partie d'un bloc, la lecture doit être fusionnée.

Comment

Utilisé comme champ de commentaire pour la variable. Il n'est pas utilisé par le système.

Behavior

Définit si la variable est du type “lecture”, “lecture/écriture”, “alarme” ou “événement”.

Blk Size

Nombre de registres dans le bloc, maximum 64 registres.
