Einführung

Die Software auf jedem Gerät Mect (TP, TPAC und TPLC), basierend auf MPX28 Karte, ist wie folgt strukturiert.

Auf TP, TPAC und TPLC-Geräten laufen zwei Anwendungen (HMI und SPS), die über eine Variablentabelle namens “Crosstable” miteinander kommunizieren

  1. HMI-Anwendung: ist die grafische Anwendung, die das Display und den Touchscreen verwaltet

  2. SPS-Anwendung: ist die SPS-Anwendung (IEC-61131-3), die die Ein- und Ausgänge mit den verschiedenen Feldbussen (Modbus RTU, Modbus TCP, CANopen, ...) verwaltet.

Die Panels werden von einem PC / Laptop mit den Software QtCreator, ATCMcontrol programmiert.

Alle diese software sind MectSuite.

Dank der hohen Integration ist es möglich, eine grafische Oberfläsche zu programmieren, ohne erfahrene Programmierer zu sein.

Das System ist offen und Sie können, falls erforderlich, komplexe Anwendungen zu estellen.

Hauptmerkmale des Systems:

  • Echtzeit

  • Fernsteuerung über VNC

  • Datenaufzeichnung

  • Online debugging von beiden HMI und SPS

  • Rezepte

  • Alarm

  • Grafiken / Trends

  • Wi-Fi und Mobile (GPRS, UMTS, …) Verbindungen [über opzionalen USB-Stick]

  • Setup () und loop (): Möglichkeit, eigene Anwendung auch mit der einzigen HMI-Umgebung (C / C ++) zu erstellen

  • Remote-Verbindung OpenVPN

  • SFTP Sicherer Server

  • Verwaltung von Protokollen:

    • Modbus TCP sowohl Client als auch Server

    • Modbus RTU sowohl Client als auch Server

    • Modbus TCP RTU sowohl Client als auch Server

    • CANopen
