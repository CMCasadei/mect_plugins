Introduction

Le logiciel de chaque appareil Mect (TP, TPAC et TPLC), basé sur la carte MPX28, est structuré comme indiqué dans la figure.

Sur les appareils TP, TPAC et TPLC s'exécutent deux applications (HMI et PLC) qui communiquent entre elles via un tableau de variables appellé“Crosstable”:

  1. Application HMI: est l'application graphique qui gère l’affichage et l’écran tactile

  2. Application PLC : est l'application API (IEC-61131-3) qui gère les entrées et les sorties avec les différents bus de terrain (Modbus RTU, Modbus TCP, CANopen, ...)

Les panneaux sont programmés à partir d'un PC / Laptop en utilisant les logiciels QtCreator, ATCMcontrol.

L'ensemble de ces logiciels constitue le MectSuite.

Grâce à la haute intégration, il est possible de programmer une interface graphique sans être des programmeurs experts.

Le système est ouvert et il est possible, si nécessaire, de créer des applications complexes.

Principales caractéristiques du système:

  • Temps réel

  • Contôle à distance via VNC

  • Enregistrement des données

  • Débogage en ligne de l’environnement IHM et de l’API

  • Recettes

  • Alarmes

  • Graphes / Tendances

  • Gestion Wi-Fi et Mobile (GPRS, UMTS, …) [via une clé USB en option]

  • Setup() & loop(): possibilité de créer votre propre application, même avec le seul environnement IHM (C / C ++)

  • Connexion à distance OpenVPN

  • Serveur sécurisé SFTP

  • Gestion des Protocoles:

    • Modbus TCP client et serveur

    • Modbus RTU client et serveur

    • Modbus TCP RTU client et serveur

    • CANopen
