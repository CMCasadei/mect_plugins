Introducción

El software en cada dispositivo Mect (TP, TPAC y TPLC), basado en la tarjeta MPX28, está estructurado como se muestra.

En los dispositivos TP, TPAC y TPLC se ejecutan dos aplicaciones (HMI y PLC) que se comunican entre sí a través de una tabla de variables llamada “Crosstable”:

  1. Aplicación HMI:es la aplicación gráfica que gestiona la pantalla y la pantalla táctil.

  2. Aplicación PLC:es la aplicación PLC (IEC-61131-3) que gestiona las entradas y salidas con los diversos buses de campo (Modbus RTU, Modbus TCP, CANopen, ...)

La programación de los paneles se realiza mediante PC / laptop utilizando los software QtCreator, ATCMcontrol.

El conjunto de estos software conforma el MectSuite.

Gracias a la alta integración es posible programar una interfaz gráfica sin ser expertos programadores.

El sistema está abierto y es posible, si es necesario, crear aplicaciones complejas.

Principales características del sistema:

  • Tiempo Real

  • Control Remoto a traves de VNC

  • Registro de datos

  • Depurar en línea tanto el entorno HMI como PLC

  • Recetas

  • Alarmas

  • Gráficos / Tendencias

  • Gestion Wi-Fi y Mobile (GPRS, UMTS, …) [a través de una memoria USB opcional]

  • Setup () y loop (): capacidad de crear su propia aplicación, incluso con el único entorno HMI (C / C ++)

  • Conexión remota OpenVPN

  • Servidor SFTP Seguro

  • Gestión de Protocolos:

    • Modbus TCP cliente y servidor

    • Modbus RTU cliente y servidor

    • Modbus TCP RTU cliente y servidor

    • CANopen
