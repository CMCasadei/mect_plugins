PLC(SPS)

ATCM control wird zum Schreiben der SPS-Anwendung verwendet.

Die SPS-Software besteht aus einer Reihe von Zyklen, die “Task”,die garantiert zu vorgegebenen Zeiten ausgeführt werden.

Jeder task kann ein “Programm” oder mehrere davon enthalten, in das der Programmierer seinen eigenen Code in seiner bevorzugten Sprache einfügt. Andere wiederkehrende Codeblöcke, die “Funktionen” und/oder “Funktionsblöcke” genannt werden, können auch innerhalb von Programmen verwendet werden.

Im Gegensatz zu Funktionen haben Funktionsblöcke folgende Eigenschaften:

  • Möglichkeit des Debuggens.

  • Verwendung von Timern.

  • mehrere Ein- und Ausgänge.

Es liegt an dem Programmierer, die Verwendung innerhalb seiner eigenen Programme zu wählen.



Das Gerät wird nicht mit dem Quellcode der SPS-Logik heruntergeladen, sondern mit einem Binärcode. Aus diesem Grund wird es nicht möglich sein, den Quellcode aus dem Produkt abzurufen.

ACHTUNG: Das SPS-Projekt muss:

  • Nennen sie sich plc

  • Sich in dem von der HMI-Umgebung erstellten Projektordner befinden (gleiche Ebene wie der Ordner “config”).
