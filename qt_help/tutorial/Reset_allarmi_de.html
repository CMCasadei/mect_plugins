Alarme anerkennen

Die Alarme werden vom Benutzer wie im Kapitel “Alarmfunktion” beschrieben verwaltet.

Durch Eingabe des folgenden Codes in die Funktion “ void loop(void)” der Seite “automation.cpp” ist es möglich, die Alarme über eine ATCM-Taste oder direkt aus dem SPS-Code anerkennen.

		if (Reset_alarms){
        pthread_mutex_lock(&alarmevents_list_mutex);
        {
            for (int i = 0; i < _active_alarms_events_.count(); i++)
            {
                if (! _active_alarms_events_.at(i)->isack)
                {
                    _active_alarms_events_.at(i)->ack = QDateTime::currentDateTime();
                    _active_alarms_events_.at(i)->isack = true;
                }
            }
        }
        pthread_mutex_unlock(&alarmevents_list_mutex);
        doWrite_Reset_alarms(false);
    }

ACHTUNG: Die Variable “Reset_alarms” ist eine Variable des “Crosstable” (Type: BIT, Behavior: Read/Write).


Alarme zurücksetzen

Um die Alarmhistorie zu löschen, müssen Sie “Go to slot...” → “clicked” auf einer ATCMbutton machen (siehen Sie Abschnit “EINSEITIGE SEITE” im Kapitel “Realisierung einer Automatisierung”) und innerhalb der Funktion den folgenden Code einfügen:

system("rm -rf /local/data/alarms/*.log");
