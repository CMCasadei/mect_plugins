Crosstable Editor functionality

  • Update table: is the main key of Crosstable: allows the insertion of user data in the Crosstable table.

  • Check: key verifies the correctness of inserted data. Led on key changes according to status:

    • Red: wrong data (application not working);

    • Yellow: changes respect previous check;

    • Green: check ok.

    When check is OK the file “Crosstable.gvl” (with plc variables) is automatically generated and must be saved in the PLC project.

  • Crosstable Import: load the “Crosstable.csv” file of another project. Imported file overwrite previous one when saved.

  • CAN import: imports the CAN variables generated by CANBuilder tool. To write the number of the row of the crosstable where you want to import the variables, press “CAN Import” and select the “ASCIIcmg.txt” file generated by CANBuilder.

  • Introduce in a row: this key insert in a semi-automatic way the variable to put in the Crosstable (table). When “Introduce in a row” is selected, a windows appears (Row number) to insert the desired row number of the Crosstable where to store the variable.

  • Previous Block: if the new variable to insert in the Crosstable must be in the same block of a previous variable it is necessary to select “Previous block” key. This function allows the creation of variables block. This key is not always visible and usable.

  • Undo: undo last action.

  • Show: this key shows in the table only the used variables. Every time the key is pressed, in the first row of Crosstable the next variable is shown.

  • Jump: insert in “Row number” the number of the row to show and press “Jump”. Table shows the desired row.

  • Move to: moves a row in a desired position. Select the row to move in the table and insert the new row number in “Row number”. Two scenarios:

    1. MOVE A ROW INSIDE AN EXISTING BLOCK

      • Select the row to move (example: row 7)

      • Insert in the key “Row number” the new position (example: position 3)

      • Selected row is moved in the new position (position 3) and is merged with block of the preceding row (in our case in the block of row 2)

    1. MOVE A ROW AND CREATE A NEW BLOCK

      • Select the row to move (example: row 7)

      • Insert in the key “Row number” the new position. WARNING: to create a new block the previous row must be EMPTY

      • Selected row is moved in the new position and a new block is created

  • Save: save current setup.

  • Save As: save current setup in a desired folder.

  • Exit: close program and ask to save changes.

  • Delete Row: deletes a Crosstable row. The row to delete must be selected by key “Select” (in the right side of the table).

  • Select: “Select” key in the table, loads the selected row in the upper side of the table where is possible to change.


ALARM / EVENT variable:

If a variable must be an alarm/event one it is necessary to define it in the “Behavior” field but it is possible only if:

  • Update” field is not “H”;

  • Type” field is equal to “BIT”;

With this setup it is possible to choose the alarm/event condition.

Variables in the drop menu are all present in the Crosstable and system variables but not those with “Update” fields equal to “H” and those with “Priority” field equal to “0”.
