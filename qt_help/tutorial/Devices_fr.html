Devices

La fenetre “Devices” fournit une vue topologique des variables utilisées dans la Crosstable du projet.


L'interface est la suivante:

Dans la fenêtre il y a 3 colonnes:

  • Name: Arborescence avec l'organisation suivante:

    • Panneau → Device (interface de communication) → Node → Priority → Block → Variables.

  • Info: Informations sur l'élément dans l'arbre:

    • Pour le Panneau, il résume le nombre de variables utilisées dans le projet, les interfaces utilisées, les paramètres Retries et Blacklist(de “System”).

    • Pour les Devices (interface) résume le nombre de variables relatives à l'appareil, les paramètres Max Block Size, Timeout et Silence (de “System”). Pour les interfaces RTU, le temps de silence minimum requis par la vitesse de transmission réglée (dans “System”) est calculé et si ce temps est suffisant (OK ou too short).

    • Pour les Nœuds, le nombre de variables gérées par le nœud.

    • Pour les Blocs, le nombre de variables dans le bloc.

    • Pour chaque Variable du bloc, la ligne de la Crosstable, le type et le comportement (R/O, RW, etc.).

  • Timings: Informations sur les temps de lecture requis pour l'élément sélectionné (uniquement pour les éléments de type RTU):

    • Pour les Devices, le temps total requis pour lire toutes les variables regroupées également par Priorité.

    • Pour les Nœuds, le temps nécessaire pour lire les variables de nœud qui sont également regroupées par priorité.

    • Pour chaque niveau de Priorité, le temps nécessaire pour lire les blocs à ce niveau de priorité.

    • Pour les Blocs, le temps nécessaire pour lire le bloc.
