PLC(API)

ATCM control est utilisée pour écrire l'application API.

Le logiciel PLC est composé d'une série de cycles appelés “Task”, dont l'exécution est garantie dans des temps prédéterminés.

Chaque task peut contenir un ou plusieurs “programme”, dans lequel le programmeur insère son propre code dans le langage de son choix. D'autres blocs de code récurrents appelés “fonctions” et/ou “blocs de fonction” peuvent également être utilisés dans les programmes.

Contrairement aux fonctions, les blocs fonction présentent les caractéristiques suivantes:

  • possibilité de débogage.

  • l'utilisation de temporisateur.

  • entrées et sorties multiples.

C'est au programmeur de choisir leur utilisation dans ses propres programmes.



Sur l'appareil n'est pas téléchargé le code source de la logique PLC(API) mais un code binaire. Pour cette raison, il ne sera pas possible de récupérer le code source depuis le produit.

ATTENTION: Le projet PLC doit:

  • S'appeler plc

  • Etre dans le dossier projet créé par l'environnement HMI (même niveau que le dossier “config”).
