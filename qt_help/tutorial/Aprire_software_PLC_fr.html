Ouvrir le logiciel PLC

Le logiciel API(PLC) est lancé via le bouton de la Crosstable.

Les variables présentes dans la Crosstable correspondent à celles du logiciel du PLC. La mise à jour de la liste des variables peut se faire de 3 façons:

  • En appuyant sur le bouton de la Crosstable

  • En faisant le Build de la partie HMI:

  • En faisant le Deploy de la partie HMI:

Dans l'environnement ATCMControl, il y aura les variables de la Crosstable qui ont été importées dans le projet PLC.

  • En étendant “Configuration” et “Resource1” on trouve les fichiers “Crosstable.gvl”, “Resource1.gvl” et les diverses Tasks.

le fichier Crosstable.gvl est le fichier qui contient toutes les variables globales d'échange entre HMI et PLC. Ce fichier ne doit être utilisé qu'en lecture pour connaître les noms des variables et ne doit jamais être modifié.

Dans le fichier Resource1.gvl toutes les variables globales du PLC sont déclarées. Dans ce fichier, vous pouvez insérer des variables dont l'environnement HMI n'a pas besoin.

il y'a 6 Tasks qui ont des fonctionnalités et des temps de cycle différents:

    • Task0_Init: ne s'exécute qu'au démarrage de l'API (PLC) et a donc une priorité absolue. Habituellement utilisé pour initialiser les variables.

    • Task1_Alarms: : a la priorité sur toutes les autres tâches. Il démarre lorsque l'alarme est déclenchée et que le programme inséré dans la tâche n'est exécuté qu'une seule fois. Le Task1_Alarms redémarre à chaque nouvelle condition d'alarme.

    • Task2_Events: : a la priorité sur toutes les autres tâches sauf Task1_Alarms. Il démarre lorsque l'événement est déclenché et que le programme inséré dans la tâche n'est exécuté qu'une seule fois. Le Task2_Events redémarre à chaque nouvelle condition d'événement.

    • Task3_Fast_10ms: cette tâche est exécutée toutes les 10ms.

    • Task4_Normal_100ms: cette tâche est exécutée toutes les 100ms.

    • Task5_Slow_1000ms: cette tâche est exécutée toutes les 1000ms.

REMARQUE: Il est conseillé d'utiliser Task4_Normal_100ms pour la plupart des applications.

Dans chaque Task,il y'a au moins un programme, par exemple si vous étendez le Task5_Slow_1000ms vous trouverez le programme Slow_Inst1 où vous pouvez implémenter, dans ce cas dans le langage ST, une partie de votre application. De cette façon, il est possible d'écrire du code à l'intérieur des programmes présents dans chaque tâche.
