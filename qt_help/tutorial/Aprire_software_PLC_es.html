Abrir el software PLC

El software PLC se inicia a través del botón de la Crosstable.

Las variables presentes en la Crosstable corresponden a las del software PLC. La actualización de la lista de variables se puede hacer de 3 maneras:

  • Pulsando el botón de la Crosstable

  • Haciendo Build de la parte HMI:

  • Haciendo Deploy de la parte HMI:

En el entorno ATCMControl estarán las variables de la Crosstable que se han importado al proyecto PLC.

  • Ampliando “Configuration” y “Resource1” encontramos los archivos “Crosstable.gvl”, “Resource1.gvl” y varios Tasks.

El archivo Crosstable.gvl es el archivo que contiene todas las variables globales de intercambio entre HMI y PLC. Este archivo sólo debe ser usado para leer y conocer los nombres de las variables y nunca debe ser modificado.

En el archivo Resource1.gvl se declaran todas las variables globales del PLC. En este archivo se pueden insertar variables que no son necesarias para el entorno HMI.

Hay 6 Task que tienen diferentes funcionalidades y tiempos de ciclo:

    • Task0_Init: se ejecuta sólo cuando se arranca el PLC y por lo tanto tiene prioridad absoluta. Normalmente se utiliza para inicializar variables.

    • Task1_Alarms: tiene prioridad sobre todas las demás tareas. Se inicia cuando se activa la alarma y el programa insertado en la Tarea se ejecuta sólo una vez. El Task1_Alarms se reiniciará cuando se produzca cada nueva condición de alarma.

    • Task2_Events: tiene prioridad sobre todas las demás tareas excepto Task1_Alarms. Se inicia cuando se activa el evento y el programa insertado en la Tarea se ejecuta sólo una vez. El Task2_Events se reiniciará cuando se produzca cada nueva condición de evento.

    • Task3_Fast_10ms: esta tarea se ejecuta cada 10ms.

    • Task4_Normal_100ms: esta tarea se ejecuta cada 100ms.

    • Task5_Slow_1000ms: esta tarea se ejecuta cada 1000ms.

NOTA: es aconsejable utilizar Task4_Normal_100ms para la mayoría de las aplicaciones.

Dentro de cada Tarea hay al menos un programa, por ejemplo al ampliar del Task5_Slow_1000ms se encuentra el programa Slow_Inst1 donde puede implementar, en este caso en el lenguaje ST, parte de su aplicación. De esta manera es posible escribir algún código dentro de los programas presentes en cada Tarea.
