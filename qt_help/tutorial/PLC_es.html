PLC

ATCM control se utiliza para escribir la aplicación de PLC.

El software del PLC está formado por una serie de ciclos denominados “Task” cuya ejecución está garantizada en unos tiempos predeterminados.

Cada task puede contener un “programa” o más de ellos, en el que el programador inserta su propio código en su lenguaje preferido. Otros bloques de código recurrentes llamados “funciones” y/o “bloques de función” también se pueden utilizar dentro de los programas.

A diferencia de las funciones, los bloques de funciones tienen las siguientes características:

  • posibilidad de depuración.

  • uso de temporizadores.

  • múltiples entradas y salidas.

Depende del programador elegir su uso dentro de sus propios programas.



El dispositivo no descarga el código fuente de la lógica del PLC sino un código binario. Por esta razón no será posible recuperar el código fuente desde el producto.

ATENCIÓN: El proyecto PLC debe:

  • Llamarse plc

  • Estar dentro de la carpeta de proyecto creada por el entorno HMI (mismo nivel que la carpeta “config”).
