Web interface functioning

Panel has a web server that allows a browser to access several web pages to verify some panel setup or to perform upload and download.

The panel IP address is displayed in the HMI in the info page in the main menu of the demo application.

To use web interface it is necessary to write panel IP address in the URL space of the browser.

To access the web interface a password is required. Passwords states the access rights to the web interface functionality. Default values are:

  • User : 111111

  • SuperUser: 222222

  • Admin: 1357924680

Logs

Log page starts web interface devoted to history management. There are 3 kinds of operation on history: update of partial history definition and, in the “Manage Logs” page the download and delete of the partial and total history.

Update LOG

Update function for partial history definition refresh the list of the partial history available on panel. The history definition must be in the CSV format and file name must begin with prefix “store”:

date

time

VARIABLE 1

…

VARIABLE N

The interface uploads one CSV file at a time, so to upload more than one definition it is necessary to perform more than one upload.

Downolad LOGS

Web interface allows a log download in this way:

  • Whole all / partial all: if you select “Total” log or one of the partial log without inserting any value in the date field all logs on the panel are downloaded.

  • Whole day / partial day: if you select Total log or one of the partial log and writing a value in the “Initial Date” field the log of the specified day is downloaded.

  • Daily total in a time interval / Daily partial in a time interval: if you select “Total” log or one of the partial log and writing a value in the “initial Date” and “Final Date” field the day logs in the specified time interval is downloaded.

  • Unique total in a time interval / Unique partial in a time interval: if you select “Total” log or one of the partial log and writing a value in the “Initial Date” “Initial Time” “Final Date” and “Final Time” fields a unique log with all stored data in the specified interval is downloaded.

Delete LOGS

Web interface allows to delete:

  • partial logs definition: select partial logs without specifying any time interval and delete

  • all logs on the system: select “Total” log without specifying any time interval and delete

  • specific day log: select total log and write a value in the “Initial Date” field.

“Initial Time”, “Final Date” and “Final Time” fields must be left blank

  • Specific time interval logs: select total log and write a value in the “Initial Date”, “Final Date”. “Initial Time” and “Final Time” fields must be left blank.

WARNING: it is not possible to delete partial logs (day or time interval) because they are generated from total logs at the download moment.

Trend

Trend page starts interface devoted to the trend management. There are 4 modes to manage trends: update, screenshot download and in the “Manage Trend” page the download and delete.

Update

Update function of family trends refresh the list of trends family on the panel. Trend families to upload must be in a zip archive. The archive has a file list each of them describes a trend type.

Manage trend

After the upload of trends as described the Manage Trend page allows to browse in the folders containing trends families. It is possible to download or delete a trend family or a series of families by multiple selection.

Alarms

Alarms page starts web interface to manage alarms.
2 type of operation are possible: download and delete of alarms in the panel.

Recipes

Recipes page starts the interface for recipes management.

3 type of operation are possible: update and, in the “Manage Recipes” page, the download and delete.

Update

Update for recipes refresh the list of recipes on the panel. Recipes to update must be in a zip archive. The archive has folders containing a recipe type each, and inside each folder there are several implementation of the recipe.

For example, if you want 2 recipes families, one for chilling and one for heating, in the archive with recipes there is:

├── cool

│ ├── cool_10

│ ├── cool_20

│ └── cool_30



└── hot

├── hot_100

├── hot_150

├── hot_200

└── hot_50

Manage recipes

After uploading the recipes as described, the Manage Recipes page allows to browse in the folder containing the recipes. It is possible to download or delete recipes on the entire family selecting the family, or on a single recipe.

LAN Config

In this page the net properties of the panel are set. Panel has only a static net configuration for which it is possible to define the IP address, the net mask (NM), the default gateway (GW) and the server dns (DNS1 and DNS2)

Date and Time

“Date and Time” page receives date and time from PC where browser runs and allows to set the values on the panel itself.

Display Settings

Display settings page allows the following:

  • Display brightness: it is possible to set a value between 0 and 100 (minimum and maximum brightness level)

  • Screensaver: it is possible to set the inactivity time interval to start the screensaver. The value must be in seconds. A 0 value disable the screensaver.

  • Touch Calibration: it is possible to calibrate the touch screen of the display. This procedure stops the HMI and PLC software and ask the user to touch some points on the display. At the end of calibration HMI and PLC automatically restart.

Manage Configuration

Allows logged user to upgrade the splash screen (splash.png) at the startup of HMI application

Manage Password

Allows user logged with password “User” or “Super User” to change its own password only. Password of “Admin” level cannot be changed. WARNING: these passwords are different from the HMI application passwords.

Passwords reset

Reset password function reset passwords of web interface to the default value. Only for “Admin” user.
