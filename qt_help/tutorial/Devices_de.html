Devices

Das Fenster “Devices” bietet eine topologische Ansicht der im crosstable des Projekts verwendeten Variablen.


Die Schnittstelle ist wie folgt:

Im Fenster gibt es 3 Spalten:

  • Name: Baumansicht mit der folgenden Organisation:

    • Panel → Device (Kommunikationsschnittstelle) → Node → Priority; → Block → Variables.

  • Info: Informationen über das Element im Baum:

    • Für das Panel fasst es die Anzahl der im Projekt verwendeten Variablen, die verwendeten Schnittstellen, die Retries und Blacklist-Parameter (aus “System”) zusammen.

    • Für die Devices (Schnittstelle) fasst die Anzahl der auf das Gerät bezogenen Variablen, die Parameter Max Block Size, Timeout und Silence (aus “System”) zusammen. Für RTU-Schnittstellen wird die minimale Stillstandszeit berechnet, die von der eingestellten Baudrate (im “System”) benötigt wird und ob diese Zeit ausreichend ist (OK oder too short).

    • Für die Knoten die Anzahl der vom Knoten verwalteten Variablen.

    • Für die Blöcken die Anzahl der Variablen im Block.

    • Für jede Variable des Blocks die Crosstable Zeile, den Typ und das Verhalten (R/O, RW, etc.).

  • Timings: Informationen über die für das ausgewählte Element erforderlichen Lesezeiten (nur für Elemente vom Typ RTU):

    • Für die Devices die Gesamtzeit, die benötigt wird, um alle Variablen zu lesen, die ebenfalls nach Priorität gruppiert sind.

    • Für die Knoten die Zeit, die zum Lesen der Knotenvariablen benötigt wird, die ebenfalls nach Priorität gruppiert sind.

    • Für jede Priorität stufe die Zeit, die benötigt wird, um die Blöcke auf dieser Prioritätsstufe zu lesen.

    • Für die Blöckendie Zeit, die zum Lesen des Blocks benötigt wird.
