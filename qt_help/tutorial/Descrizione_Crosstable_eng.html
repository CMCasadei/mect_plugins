Description of Crosstable fields

The Crosstable is divided into several sections, some modifiable from the user and others blocked and precompiled with the variables of the device.

“Retentive”section, the variables in this section do not lose their value after turning the Mect device off and on again.

“Non Retentive”section, a large section available for all the variables of your project that do not need to be “remembered” after the device has been switched off.

“Diagnostic”, “Local I/O” and “System” sections are partially editable by the user and can be used in your projects, as well as allowing you to move the I / O, they provide important information on the internal operation of the Mect device.


Field

Description

Priority

Remote devices data exchange enable.

- 0: Disabled;

- 1: High priority enable; preferably for variables with “PLC” protocol.

- 2: Medium priority enable; preferably for variables with “RTU” protocol.

- 3: Low priority enable. preferably for variables with “TCP” protocol.

Update

It defines the type of data exchange enabled with the remote devices.

-H: the variable is updated only if it appears in the current page;

-P: the variable is always updated;

-S: the variable is always updated and logged slowly with time “Slow Log Period” (System Editor parameter);

-F: like “S” but quickly with time “Fast Log Period” (System Editor parameter);

-V: variable is stored only on change;

-X: variabile is stored when logShot() function is called.

Var.Name

Name of the variable (30 char max and different by “Text”).

Type

The following types are defined:

-BIT: boolean;

-BYTE: byte;

-BYTE_BIT: select a BIT of a byte;

-WORD_BIT: select a BIT of a word (16 bit);

-DWORD_BIT: select a BIT of a double word (32 bit);

-UINT: unsigned integer 16 bit;

-UINTBA: unsigned integer 16 bit swapped byte;

-INT: integer 16 bit;

-INTBA: integer 16 bit swapped byte;

-UDINT: unsigned integer 32 bit;

-UDINT....: unsigned integer 32 bit swapped byte;

-DINT: integer 32 bit;

-DINT....: integer 32 bit swapped byte;

-REAL: real;

-REAL...: real swapped byte.

Decimal

For integer or real variables is the number of decimal values of the variables. For variables BYTE_BIT, WORD_BIT or DWORD_BIT is the bit position starting from 1.

Protocol

Type of protocol to be used with the remote device:

-PLC: variable for exchange between PLC and HMI

-RTU: variable on a remote device on Modbus RTU

-TCP: variable on a remote device on Modbus TCP

-TCPRTU: variable on a remote device on Modbus TCPRTU

-CANOPEN: variable on a remote device on CANopen

-RTU_SRV: variable for a master Modbus RTU

-TCP_SRV: variable for a master Modbus TCP

-TCPRTU_SRV: variable for a master Modbus TCPRTU

-MECT: variable on MECT protocol

IP Address

IP address for modbus TCP and TCPRTU, for example: 192.168.0.42

Port

For NON PLC variables is the communication port to use: it depends on used device.

Node ID

Address of the remote device node. In case of 0 address the writing is Broadcast.

Register

Modbus register address.

Block

Address of the first register of the block. For registers belonging to the same block only one reading request is issued.

Comment

Variable comment field. Not used by system.

Behavior

Defines if variable is “Read”, “Read/Write”, “Alarm” or “Event”.

Blk Size

Number of registers available in the block. Max 64 registers.
