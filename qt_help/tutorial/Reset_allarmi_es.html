Reconocer de alarmas

El usuario gestiona las alarmas como se describe en el capítulo “Funcionamiento de las alarmas”

Introduciendo el siguiente código en la función “ void loop(void)” de la página “automation.cpp” es posible reconocer las alarmas mediante un botón ATCM o directamente desde el código plc.

		if (Reset_alarms){
        pthread_mutex_lock(&alarmevents_list_mutex);
        {
            for (int i = 0; i < _active_alarms_events_.count(); i++)
            {
                if (! _active_alarms_events_.at(i)->isack)
                {
                    _active_alarms_events_.at(i)->ack = QDateTime::currentDateTime();
                    _active_alarms_events_.at(i)->isack = true;
                }
            }
        }
        pthread_mutex_unlock(&alarmevents_list_mutex);
        doWrite_Reset_alarms(false);
    }

ATENCIÓN: la variable “Reset_alarms” es una variable de la “Crosstable” (Type: BIT, Behavior: Read/Write).


Reinicio de alarmas

Para eliminar el historial de alarmas, debe hacer un “Go to slot...” → “clicked” en un ATCMbutton (ver la sección “PÁGINA ÚNICA” en el capítulo “Realización de una automatización”) e insertar, dentro de la función, el siguiente código:

system("rm -rf /local/data/alarms/*.log");
